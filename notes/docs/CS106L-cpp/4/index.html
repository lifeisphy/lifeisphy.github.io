    Link error #

    链接时，每一个程序文件都将被单独处理，这就允许一个文件中使用声明但未实现的函数。只有当链接过程结束，该函数依然未定义时才会报出错误。链接过程的具体细节这里不再阐述。

    举个例子：下面的例子可以实现字符串的小写转换：

    #include <iostream>
#include <string>
#include <cctype>
string ConvertToLowerCase(string input); // prototype
int main(){
  string myString = "THIS IS A STRING!";
  cout<<ConvertToLowerCase(myString);
  
}
string ConvertToLowerCase(string& input){
  for(int k = 0; k<input.size(); ++k){
    input[k] = tolower(input[k]);
  }
  return input;
}

    但该程序会出现链接错误。因为定义参数表类型与声明类型不相符：string& input声明的字符串为按引用传递，而声明中为按值传递。这就导致它们被视为两个不同函数，因而导致ConvertToLowerCase未被实现。

    Preprocessing commands #

    1. #include <xxx> or "xxx.h"：直接将文件内容插入在当前位置

    2. #define phrase replacement：将文件中所有的phrase替换为replacement。如果是一整个表达式，需要加括号以指定计算优先级；replacement结尾不要加分号 #define的变式：#define phrase，此时replacement为空字符串

    3. #if statement,#elif statement,#else statement,#endif:用于条件控制 其中的statement子句可以是：

    #if MY_CONST> 137
#if MY_CONST*42 == MY_CONST 
#if sqrt(MY_CONST)<4 // illegal ,cannot call function sqrt
#if MY_CONST == 3.14 //illegal, only integers allowed

    也可以使用defined谓词：

    #if defined(MY_CONSTANT) // true/false,equivalent to #ifdef MY_CONSTANT
#if !defined(MY_CONSTANT) // #ifndef MY_CONSTANT

    使用预处理指令做注释：

    #if 0
xxx
#endif

    和使用/**/作为注释不同的是，预处理指令本身可以嵌套（nested），因而可以避免普通注释中的一些问题。

    Macros: #

    预处理宏和C++函数除了接口相似外，几乎没有其他相似之处。C++ functions present code that executes at runtime to manupulate data, while macros expand out into newly-generated C++ code during preprocessing. #define macroname(param1,param2,...,paramN) macrobody

    例子： #define PLUS_ONE(x) ((x)+1) 如果写int x = PLUS_ONE(137)，则它会被替换成int x = ((137)+1) 预处理宏没有返回值，仅仅是将宏定义的内容进行扩展。

    在macros和C++ functions之间选择时，应尽可能选择后者，因为它更加安全，更不易错，可读性更强

    Inline functions #

    对于MAX这样的简单函数，使用宏定义函数有一个优点，那就是代码将在行内展开，程序运行时不必为之分配额外的栈空间，因而运行更快，效率更高。为了将这种特性应用在函数上，C/C++发明了inline关键字，若函数使用了这个关键字，编译器将在行间展开这个函数。

    inline int Max(int one,int two){
  return one> two? one: two;
}

    inline对编译器而言只是一个建议，因为行间展开可能很困难/消耗很大。编写排序函数时，常常使用这个关键字提升性能。

    Special preprocessor values #

    __DATE__ and __TIME__ contain string representations of the date and time that the program was compiled,__LINE__ and __FILE__ gives the line number and filename of the code respectively. See special.cpp

    String manipulation functions #

    由于宏定义直接操纵源码，因而能实现一些普通函数无法实现的功能。

    1. stringizing operator: #

    for example:

    #define PRINTOUT(n) cout<< #n <<" has value " << (n) << endl
int x = 137;
PRINTOUT( x* 42); -> //cout<<"x * 42"<<" has value "<<(x * 42) <<endl;

    It preserves the string variable name in the final program.

    1. string concantenation operator: ## 连接前后字符串. It takes string value of a parameter and concatenates it with another string. For example, consider the following macro:
    #define DECLARE_MY_VAL(type) type my_##type
DECLARE_MY_VAR(int); //-> int my_int;

    Advanced use: X macro calls #

    color.cpp和 color.h是使用这个技巧的一个例子。在这个例子中，Color是一个枚举类型，我们希望写两个函数：给定颜色，返回颜色的字符串；给定颜色，返回对应的反色。所有颜色的反色信息存储在color.h，这是一系列未给出宏定义的宏函数DEFINE_COLOR。这个函数只在实现两个函数时进行定义，且可以给出不同的定义。
